# Contributing

Thanks for your interest in contributing! This is a small personal project, so there's no formal process — just a few things to keep in mind.

## Getting started

Follow the local development steps in the [README](README.md).

## Making changes

1. Fork the repo and create a branch from `main`.
2. Make your changes — see [guidelines.md](.ai/guidelines.md) for code conventions and architecture notes.
3. Run the tests and linters before submitting (see below).
4. Open a pull request with a brief description of what you changed and why.

There's no issue template or strict PR format. A sentence or two explaining the change is fine.

## Tests and linting

**PHP code style (Laravel Pint):**
```bash
./vendor/bin/pint
```

**JavaScript/TypeScript formatting (Prettier):**
```bash
npx prettier --write resources/
```

## Questions

If you're unsure about something, just open an issue or a draft PR and ask. There's no silly question.
